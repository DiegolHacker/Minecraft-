Preguntas Lab 1

¿Cuál es la diferencia entre Internet y la World Wide Web?

El internet es una red de computadoras conectadas que transmiten información entre sí, la WWW es una serie de páginas web que existen encima del internet.

¿Cuáles son las partes de un URL?

Protocolo, subdominio, dominio, TLD, carpeta, página, etiqueta.

¿Cuál es el propósito de los métodos HTTP: GET, HEAD, POST, PUT, PATCH, DELETE?

Get: Recibir información del servidor.

Head: Recibir solo el encabezado de la página.

Post: Subir información al servidor.

Put: Actualizar un dato del servidor o crear un dato nuevo.

Patch: Actualizar un dato de forma parcial.

Delete: Eliminar información.

¿Qué método HTTP se debe utilizar al enviar un formulario HTML, por ejemplo cuando ingresas tu usuario y contraseña en algún sitio? ¿Por qué?

El método post ya que este método cifra en automático los datos por lo que son seguros y tiene una robustes y capacidad suficiente para enviar toda la informaicón.

¿Qué método HTTP se utiliza cuando a través de un navegador web se accede a una página a través de un URL?

Utiliza el método Get.

Un servidor web devuelve una respuesta HTTP con código 200. ¿Qué significa esto? ¿Ocurrió algún error?

Que todo salió bien, el servidor recibió la solicitud y la proceso de manera correcta.

¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 404? ¿Por qué?

Sí debido a que es un error que aparece cuando un contenido no existe, y si un usuario común lo recibe es debido a una url desactualizada que lleva a un contenido eliminado.

¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 500? ¿Por qué?

Sí debido a que es un error interno del servidor, y aunque el contenido existe y es accesible por el usuario no puede ser mostrado.

¿Cuáles son las diferencias principales entre HTML 4 y HTML5?

HTML5 cuenta con métodos que dan un formato más entendible, como header, article, etc. También cuenta con mejores formularios con espacios dedicados a cada tipo de dato, como email que produce un cuadro de entrada sólo para correos electrónicos. La compatibilidad con celulares es mucho más amplia, y cuenta con tecnologías de forma nativa como reconocimiento de voz y contenido multimedia a más de 60 fps.

De estructura existen th (Table Head), tr (Table Row), y td (Table data) que permiten llenar las celdas de una tabla con información, y de estilo thead, tbody, tfooter que determinan en que zona de la tabla se encuentran las celdas.

Se debe evitar ya que vuelve al código difícil de entender y debugear ya que deja de ser intuitivo que valor de las propiedades debe mostrarse, solo se debe usar en casos completamente necesarios como cuando está prohibido editar el código original y debemos añadir formato a una página ya existente.

Como ingeniero de software, ¿cuál es tu recomendación al elegir las unidades de un propiedad de estilo entre %, px y pt?

Depende completamente del contexto donde el sitio web vaya a ser utilizado. Si la resolución es fija, por ejemplo que sea la página de bienvenida para un dispositivo como una consola portátil, es una opción viable tener una unidad absoluta como lo son los pixeles pues las condiciones de visualización no cambiaran, pero si el tamaño de las pantallas donde se verá el contenido pueden cambiar, es mucho más recomendable usar unidades relativas para no perder la noción del espacio y las proporciones de la página, evitando que se vea mal.

¿Por qué el uso de una versión minimizada del CSS mejora el rendimiento del sitio?

Debido a que es una versión optimizada que cumple con las mismas funciones que el código original pero eliminando las partes innecesarias, como comentarios o indentación, lo que lo hace más rápido de correr en un servidor, y debdio a que en una página web puede haber miles de lineas para darle estilo estas optimizaciones se hacen más notorias.

¿Por qué es una buena práctica usar JavaScript para checar que sean válidos los inputs de las formas antes de enviar los datos al servidor?

Para asegurarnos que los datos se almacenen correctamente en la base de datos, sin tipos de dato incorrecto o datos truncados por ser muy largos, para saber que los podremos operar de las formas correctas, para poder decirle al usuario en qué se equivocó, y por seguridad para no correr código arbitrario.

Desactivando js en el navegador, inyectando código en los formularios, mandando solicitudes al servidor directamente, editando el código html o js público.

Si te puedes saltar la seguridad de las validaciones de JavaScript, entonces ¿por qué la primera pregunta dice que es una buena práctica?

Debido a que puede dar retroalimentación inmediata a los usuarios sobre si los datos que ingresaron son válidos o correctos, lo que mejora su experiencia. También puede mejorar la velocidad de respuesta al limitar la cantidad de información que se envía y procesa a solo la necesaria.

Preguntas Lab 13

¿Qué beneficios encuentras en el estilo MVC?

Facilita la legibilidad y el mantenimiento del código al tener el código agrupado y separado según sus funciones y objetivos. Permite reutilizar código de manera más eficiente y ordenada. Facilita el trabajo en equipo al establecer lugares bien definidos para cada tipo de código. Permite agregar más partes a la aplicación sin afectar a las que estaban cosntruidas previamente.

¿Encuentras alguna desventaja en el estilo arquitectónico MVC?

Es un poco más difícil de seguir en etapas iniciales del desarrollo, ya que es más incomodo separ tu código cuando se compone de muy pocas lineas. También puede haber casos donde reutilices código y al querer modificar sola una parte de la aplicación acabes modificando otras.

Preguntas Lab 17

¿Qué ventajas tiene escribir el código SQL únicamente en la capa del modelo?

Que se separa la parte de la creación de la base de datos, la lógica para llenar las tablas, y la lógica para hacer las consultas, lo que permite un mayor nivel de complejidad y control.

¿Qué es SQL injection y cómo se puede prevenir?

Es cuando un usuario pone una query de SQL como entrada donde solo deberían de haber datos, lo que hace que pueda alterar las tablas y sus datos cómo quiera. Se puede prevenir sanitizando las entradas de datos para que se guarden como string y no se puedan ejecutar.

Preguntas Lab 18

¿Qué otras formas de autentificación existen?

Autentificación de 2 factores: El usuario requiere de un método adicional además de su contraseña, por ejemplo un código o token generado en una aplicación o un mensaje de texto enviado al momento.

Autenticación basada en certificados: Cada usuario tiene un certificado digital único que se utiliza para firmar y cifrar la comunicación con el sistema.

Autenticación biométrica: Se basan en una característica física del usuario leída a través de sensores para determinar si se trata del usuario, por ejemplo huellas dactilares, retina, reocnocimeinto facial, etc.

Preguntas Lab 19

¿En qué consiste el control de acceso basado en roles?

Es un modelo de seguridad que se usa para restringir el acceso a información o funciones en un sistema dependeindo de los roles que tengan asignados los usuarios, que parten de sus responsabilidades dentro de una organización.

Investiguen y describan 2 sistemas, uno que aplique RBAC y uno que no. Realicen un análisis de las ventajas y desventajas de cada uno con respecto al control de acceso.

Microsoft Active Directory es un sistema para manejar directorios en una red de computadoras implementado en Windows e implementa RBAC a través de definir grupos de seguridad, o roles, con sus respectivos permisos y luego los usuarios son añadidos a estos grupos.

Como ventajas tiene que es un sistema centralizado y escalable, por lo que es fácil administrar los permisos de una gran cantidad de usuarios y modificar sus permisos en masa, esto también reduce el riesgo de cometer un error al ser casi imposible asignar un permiso que un usuario no debería tener sin un rol específico para ello.

Como desventajas es que puede ser más dificil establecer el sistema en un inicio, ya que puede ser necesario crear muchos roles que muy pocos usuarios usen. También es muy fácil hacer que un usuario tenga más permisos de los que necesita al asignarlo a un rol que no se ajusta totalmente a sus responsabilidades.


Para manejar los permisos de acceso a archivos en Linux tradicionalmente se asignan directamente estos permisos a cada usuario, en lugar de a un rol.

Esto tiene como ventaja que es un proceso mucho más sencillo y directo cuando solo existen muy pocos usuarios y queremos ser muy específicos en cuanto a que archivos pueden acceder, pues no se necesita de crear ni asignar roles.

De desventaja tiene que al querer escalar el sistema e incluir muchos usuarios se puede volver mucho m;as tardado el estar asignando permisos uno por uno para cada usuario. Además existe un mucho mayor riesgo de errores humanos al poder asignar un permiso que el usuario no debería tener
